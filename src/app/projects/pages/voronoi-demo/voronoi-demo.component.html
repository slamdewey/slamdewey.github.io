Voronoi Tessellation

A Voronoi diagram partitions a plane into cells, one per seed point, where every pixel in a cell is closer to its seed than to any other. This page walks through building a Voronoi tessellation from scratch, adding noise-perturbed boundaries and Lloyd relaxation — the foundation for realistic tectonic plate generation in the world-gen system.

1. Seed Placement

Everything starts with placing seed points on the plane. We need deterministic placement so the same input seed always produces the same diagram. A simple xorshift32 PRNG does the job — it's fast, stateless, and fits in a few lines:

Each seed gets a random (x, y) position within the map dimensions. With only a handful of seeds, the placement looks sparse. Increase the count and the plane fills up — but notice how some clusters form while other areas are empty. That unevenness is what Lloyd relaxation will fix later.

2. Nearest-Neighbor Assignment

With seeds placed, we assign every pixel to its nearest seed. This is the core of the Voronoi diagram: for each pixel, measure the distance to every seed, and pick the closest one. We use linear distance (with sqrt) so that noise amplitude directly corresponds to pixel displacement of boundaries.

The assignment loop finds the two closest seeds for each pixel. We track the second-closest because we'll need it later for boundary detection. The noise perturbation (covered in section 4) is applied here too — each seed has its own noise field so every boundary gets independent distortion. The result is a cell map: every pixel tagged with the index of its owning seed.

Notice how the cell edges are perfectly straight lines — each boundary is the perpendicular bisector between two seeds. The cells have wildly different sizes because the random seed placement is uneven.

Raw Voronoi — no relaxation, no noise. White dots are seed points.

3. Lloyd Relaxation

Random placement creates uneven cells. Lloyd relaxation fixes this by iteratively moving each seed toward the centroid (center of mass) of its cell, then recomputing the diagram. We use partial relaxation — each iteration moves the seed only halfway toward its centroid rather than jumping all the way there. This gives gradual convergence over several iterations instead of snapping to a near-grid after just one pass. After 3–5 iterations the cells are fairly uniform — a much better starting point for tectonic plates.

Drag the slider to see how each iteration redistributes the seeds. At 0 iterations you see the raw random placement. By 3–5 iterations the cells are fairly uniform. Going beyond that gives diminishing returns.

4. Noisy Boundaries

Pure Voronoi cells have perfectly straight edges — not very natural looking. To create organic, irregular boundaries (like real tectonic plate edges), we perturb the distance metric with OpenSimplex noise. Each seed samples its own noise field (offset in noise space by seed index) so that every boundary between any pair of seeds gets independently perturbed:

Amplitude controls how far the boundaries shift (in pixels). Frequency controls how jagged or smooth the edges are — low frequency gives broad sweeping curves, high frequency gives fractal-like detail. Together they create edges that look like real geological boundaries.

Set amplitude to 0 to see the clean geometric edges, then increase it to watch them warp.

6. Boundary Detection

For tectonic plate simulation, we need to know where plate boundaries are and how strong they are. During cell assignment we already track both the nearest and second-nearest seed distances. The ratio between them tells us how close a pixel is to a boundary:

When a pixel's nearest and second-nearest seeds are almost equidistant, it's right on the boundary (strength → 1). Deep in a cell's interior, the second seed is much farther away (strength → 0). This gradient is what we'll use to drive mountain range intensity, trench depth, and fault line effects during plate simulation.
