Voronoi Tessellation

A Voronoi diagram partitions a plane into cells, one per seed point, where every pixel in a cell is closer to its seed than to any other. This page walks through building a Voronoi tessellation from scratch — seed placement, nearest-neighbor assignment, Lloyd relaxation, cylindrical wrapping, and extracting the cell adjacency graph that downstream simulations operate on.

1. Seed Placement

Everything starts with placing seed points on the plane. We need deterministic placement so the same input seed always produces the same diagram. A simple xorshift32 PRNG does the job — it's fast, stateless, and fits in a few lines:

Each seed gets a random (x, y) position within the map dimensions. With only a handful of seeds, the placement looks sparse. Increase the count and the plane fills up — but notice how some clusters form while other areas are empty. That unevenness is what Lloyd relaxation will fix later.

2. Nearest-Neighbor Assignment

With seeds placed, we assign every pixel to its nearest seed. This is the core of the Voronoi diagram: for each pixel, measure the distance to every seed and pick the closest one.

The assignment loop is straightforward — iterate over every seed, track the closest, and tag the pixel with that seed's index. The result is a cell map: every pixel tagged with the index of its owning seed.

Notice how the cell edges are perfectly straight lines — each boundary is the perpendicular bisector between two seeds. The cells have wildly different sizes because the random seed placement is uneven.

Raw Voronoi — no relaxation. White dots are seed points.

3. Lloyd Relaxation

Random placement creates uneven cells. Lloyd relaxation fixes this by iteratively moving each seed toward the centroid (center of mass) of its cell, then recomputing the diagram. We use partial relaxation — each iteration moves the seed only halfway toward its centroid rather than jumping all the way there. This gives gradual convergence over several iterations instead of snapping to a near-grid after just one pass. After 3–5 iterations the cells are fairly uniform — a much better starting point for tectonic plates.

Drag the slider to see how each iteration redistributes the seeds. At 0 iterations you see the raw random placement. By 3–5 iterations the cells are fairly uniform. Going beyond that gives diminishing returns.

5. Cell Adjacency Graph

A pixel grid tells us which cell each point belongs to, but for simulation we need to know which cells are neighbors and how much boundary they share. We scan the cell map for adjacent pixels that belong to different cells, building an edge list and per-cell neighbor lists:

Building the graph is a single pass over the pixel grid. For each pixel, we check its right and bottom neighbors — if they belong to a different cell, we record (or increment) an edge between those two cells:

The result is a compact graph where each cell knows its neighbors and the length of each shared boundary. The white lines in the visualization connect adjacent cell seeds — this is the dual graph of the Voronoi diagram, and it's the structure that downstream simulation (like tectonic plates) would operate on.
